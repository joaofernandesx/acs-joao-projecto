1 - Introdução

 

               Actualmente é utilizada com grande sucesso como ferramenta de apoio académico em programação de computadores, uma linguagem funcional conhecida por Tartaruga em Movimento. O que este trabalho aponta desenvolver, é um compilador que processe uma junção da linguagem Logo com linguagem LISS, linguagem essa concebida também para fins pedagógicos, dando origem então à linguagem LogoLISS.

Neste relatório falar-se-á de análise e especificação do problema, incluindo todos os requisitos necessários para a execução do programa e posteriormente, serão apresentadas todas as estruturas de dados e algoritmos utilizados.

Serão também explicitados código e testes efectuados durante a resolução do problema.

 

2 - Análise e desenvolvimento

 

2.1  Organização da máquina virtual

 

A máquina virtual é baseada em Stacks. É composta por uma stack de execução, por uma stack de chamadas, por uma zona de código, e por duas heaps e de quatro registos.

A stack de execução contém valores, que podem ser inteiros, reais ou endereços. As duas heaps contêm, respectivamente, cadeias de caracteres (strings) e blocos estruturados. Cada um destes tipos de dados é referenciado por endereços. Cada bloco estruturado contém um certo número de valores e um endereço pode apontar para quatro tipos de informação, informação essa que poderá ser código, a stack, um bloco estruturado ou uma string.

Existem três registos que permitem o acesso a diferentes partes da stack, dados pelo nome de sp (stack pointer), fp (frame pointer), e gp. O registo sp, aponta para o topo corrente da stack, ou seja, aponta para a primeira célula livre da pilha. O registo fp, aponta para o endereço de base das variáveis locais. O registo gp contem o endereço de base das variáveis globais.

A máquina possui ainda um registo pc, que aponta para a instrução corrente da zona de código por executar. A stack de chamada contém pares de apontadores (i, f ). O endereço ‘i’ guarda o registo de instrução pc e ‘f’ o registo fp.

 

2.2  Instruções da máquina virtual

As instruções são designadas por um nome e podem aceitar um ou dois parâmetros, e estes parâmetros podem ser, constantes inteiras, constantes reais, etiquetas simbólicas apontando para uma zona de código ou cadeias de caracteres delimitadas por aspas. Estas cadeias de caracteres seguem as mesmas regras de formatação que as cadeias da linguagem C, como é o caso dos caracteres especiais como \", \n ou \\.

 

2.3  Estruturação do problema

 

Numa primeira instância foi necessário reconhecer as palavras/expressões reservadas da linguagem, e de seguida, foi necessário converter as palavras/expressões da linguagem LogoLISS em código assembly. Para tal, primeiro foi criado um dicionário em lex que, para além de reconhecer todas as palavras reservadas, também reconhece caracteres especiais necessários à interpretação de comandos, reconhecimento inteiros e reais para além de caracteres fora das palavras-chave. O resultado do dicionário é enviado para o parser e este juntamente com a ajuda da linguagem de programação C, vai permitir realizar cálculos aritméticos, e manuseamento de variáveis, para darmos como output final o assembly respectivo a cada expressão reconhecida pelo dicionário. Podia-se simplesmente ignorar a expressão, ou então gerar código assembly correspondente a essa mesma expressão.

 

2.4  Estrutura do Dicionário LogoLISS

 

A LogoLISS permite manusear escalares, bem como arrays do tipo inteiro, na forma de constantes e de variáveis. Utiliza ainda os movimentos da Tartaruga num plano de coordenadas cartesianas. O dicionário respectivo LogoLISS reconhece então uma estrutura do tipo:

 

PROGRAM               identifier                     '{' Body '}'

 

PROGRAM, será então a TAG que identifica o inicio da codificação, seguindo-se de um identificador que dará um nome ao programa. Depois dentro da notação de chavetas é então colocado todo o corpo da aplicação, o body. Este é composto por uma definição obrigatória de dois componentes: Declarações (DECLARATIONS) e operações (STATEMENTS).

 

DECLARATIONS    declarações     STATEMENTS          operações

 

A utilização de variáveis que não foram declaradas inicialmente não é permitido, inclusive redeclarações, e a declaração de variáveis tem de ser feita no inicio do programa. A TAG DECLARATIONS marca então o início da zona do código onde serão declaradas todas as variáveis do programa. Todas as declarações usam a notação:

 

Variável -> Tipo

Podendo também ser inicializadas da seguinte maneira:

Variável = valor -> Tipo

 

De seguida a TAG, STATEMENTS, identifica o inicio da codificação, logo a partir desta zona, não serão mais permitidas nenhumas declarações, sendo apenas efectuadas operações de quatro tipos: comandos Tartaruga, atribuições, instruções condicionais e instruções iterativas.

 

 

Assim, a estrutura do dicionário da linguagem LogoLISS utilizada no

Programa é a seguinte:

 

Comandos Tartaruga // Atribuições // InstruçõesCondicionais // InstruçõesIterativas

 

2.5  Estruturas de dados

 

Sendo a linguagem C uma linguagem imperativa, foi necessário recorrer a várias estruturas de dados de forma a permitir uma correcta conversão das instruções para código máquina.

 

typedef struct stack

{

            char *name;

            char type;

            int start;

            int end;

            struct stack *next;

} *STACK;

 

Sempre que uma variável é adicionada à pilha é simultaneamente adicionada a esta lista, ou seja, é uma lista ligada em que cada nodo corresponde a uma variável global declarada na stack. O campo type serve para fazer as várias validações, de forma a evitar incoerência na atribuição de dados. O campo start e end identificam o intervalo de posições que a variável ocupa na stack, pois os arrays ocuparem várias posições na memória. A variável STACK é então o que identifica a variável em questão.

 

 

Outra das estruturas de dados utilizada é a estrutura VAR.

 

typedef struct var{

            char *name;   

            char type;

            int intValue;    //valor inteiro

            char *strValue;           //valor strings

            int filledPos;   //posições cheias do array

            int arrValue[50];//valor do array

} VAR;

 

Quando é declarada uma variável é criada uma estrutura destas. A variável é alocada num dos campos da estrutura, dependendo do tipo dela. Entre as várias camadas do parser é passado o apontador da estrutura, permitindo assim a realização de código, atribuição de valores, ou até mesmo a declaração de variáveis. Todas estas operações dependem desta estrutura, pois todas elas são centralizadas na variável, e esta estrutura representa a variável.

Ao correr este ‘storen’ o valor 3 será guardado na posição 6 da stack que corresponde á variável arr[1].
